#### ```nmap <IPアドレス>```  
ターゲットとなるIPアドレスに対して、「よく使われるポート（上位1,000個）」をスキャンします。  
そのポートが「開いているか（Open）」「閉じているか（Closed）」「ファイアウォール等で遮断されているか（Filtered）」を調査し、稼働している大まかなサービスを特定します。

#### ```-sV``` （サービス/バージョン検出）（```nmap```のオプション）  
開いているポートで動いている**具体的なサービス名**や**ソフトウェアのバージョン**を特定しようと試みます。

#### ```-sC``` （デフォルトスクリプトの実行）（```nmap```のオプション）  
Nmapに組み込まれている「NSE（Nmap Scripting Engine）」というスクリプト機能のうち、**安全で有益なデフォルトのスクリプト群**を実行します。

よく ```-sV``` と ```-sC``` はセットで ```-sC -sV```（または ```-sCV```）として使われます。これにより、ターゲットの情報をかなり詳細に丸裸にできます。

#### ```-p <ポート番号>``` （ポートの指定）（```nmap```のオプション）  
スキャンするポート番号を個別に指定します。

#### ```-p-``` （全ポートスキャン）（```nmap```のオプション）  
存在するすべてのポート（1番〜65535番）を対象にスキャンします。

#### ```curl <URL>```
オプションを付けずに実行すると、指定したURLから返ってくる「レスポンスの本体（ボディ/Body）」をそのまま画面に出力します。
- **ウェブサイト（HTML）の場合**  
そのページのHTMLソースコードがすべて表示されます。ブラウザで「ページのソースを表示」したときと同じ内容です。
- **API（JSONやXMLなど）の場合**  
サーバーが返す生のデータ（JSON形式のテキストなど）が表示されます。  

#### ☆わかること
- サーバーから正しいHTMLやデータが返ってきているか（Webページの中身の確認）  
- APIが期待通りのデータを返しているか（プログラムの開発やデバッグ）  

#### ```-I```
##### 1. HTTPステータスコード（一番重要な情報）  
  - ```HTTP/1.1 200 OK``` ：正常に通信できている。

- ```HTTP/1.1 301 Moved Permanently``` ：別のURLへ転送（リダイレクト）されている。

- ```HTTP/1.1 404 Not Found``` ：ページが存在しない。

- ```HTTP/1.1 500 Internal Server Error``` ：サーバー側でエラーが発生している。

##### 2. サーバーの種類とバージョン (```Server:```)
- ```nginx```, ```Apache```, ```Cloudflare``` など、Webサーバーに何が使われているかがわかります。

##### 3. コンテンツの種類 (```Content-Type:```)
- ```text/html; charset=UTF-8```（HTMLページ）、```application/json```（APIデータ）など、返ってくるデータの形式がわかります。

##### 4. データのサイズ (```Content-Length:```)
- ファイルの容量（バイト数）がわかります。

##### 5. キャッシュの設定 (```Cache-Control:```)
- ブラウザやプロキシサーバーに、このデータをどれくらいキャッシュ（一時保存）させてよいかの指示がわかります。

```bash
gobuster dir -u http://10.144.142.133 -w /usr/share/wordlists/dirbuster/directory-list-2.3-small.txt -x php -x php
```
について以下で説明

#### ```gobuster dir```
- **意味：** ```gobuster``` というツールの 「**ディレクトリ/ファイル探索モード（dir）**」 を使う、という宣言です。
- ※Gobusterには他にも、DNSのサブドメインを探すモードや、S3バケットを探すモードなどがあります

#### ```-u http://10.144.142.133```
- **意味：** 調査対象（ターゲット）の **URL** を指定しています。

- ここでは ```http://10.144.142.133``` というIPアドレスのWebサイトが標的です。

#### ```-w /usr/share/wordlists/...```
- **意味：** 探索に使う **単語リスト（Wordlist）のパス** を指定しています。

- ここでは、Kali Linuxなどに標準で入っている ```dirbuster``` 用の有名な辞書ファイル（smallサイズ）を使っています。

- ```何が起きるか：``` このテキストファイルには、```admin```、```images```、```login```、```secret``` といった、Webサイトでよく使われる単語が数万行にわたって詰まっています。Gobusterはこれを上から順に読み込んで、ターゲットURLの後ろに合体させてアクセスを試します。

  - ```http://10.144.142.133/admin``` ➔ 存在する？（200 OK や 403 Forbidden が返ってきたら「発見！」と表示）

  - ```http://10.144.142.133/images``` ➔ 存在する？

  - ```http://10.144.142.133/xyz123``` ➔ 存在しない？（404 Not Found なら無視）

#### ```-x php -x php```
- **意味：** 探すファイルの **拡張子（エキステンション）** を指定しています。

- 単語リストの言葉の末尾に、```.php``` を付けたパターンも同時に探してくれます。

  - 例：単語リストに ```login``` とあれば、```http://10.144.142.133/login.php``` もチェックします。

- 💡 補足： 質問のコマンドでは ```-x php``` が2回重複して書かれています。動作上は1回書けば十分（```.php``` を探す）なので、おそらくタイポ（打ち間違い）か、コピペ時のミスだと思われます。もし ```.php``` と ```.txt``` の両方を探したい場合は、```-x php,txt``` と書くか、```-x php -x txt``` と指定します。

#### このコマンドで何がしたいのか？  
Webサイトの管理者が「一般の人には見せたくない（リンクを貼っていない）けど、サーバー上には置いてある秘密のページやファイル」を見つけ出すために使います。  
- **例：** ```http://10.144.142.133/config.php```（設定ファイル）や ```http://10.144.142.133/backup/```（バックアップフォルダ）など
セキュリティコンテスト（CTF）やペネトレーションテスト（侵入テスト）の初期段階で、ターゲットのWebサイトにどんなページが隠されているかを網羅的にハックする際の「超定番」コマンドです。
